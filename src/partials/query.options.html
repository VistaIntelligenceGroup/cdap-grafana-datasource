Metrics data within CDAP is identified by a combination of context and name.

  • CONTEXT
    • A metrics context consists of a collection of tags.
    • Each tag is composed of a tag name and a tag value.
  • METRICS
    • A name generated by CDAP, and pre-pended withsystem
    • Or a name set by a developer when writing an application, which are pre-pended with user.
  • GROUP BY
    • This optional setting defines a list of tags whose values are used to build multiple timeseries.
    • All data points that have the same values in tags specified in the GROUP BY setting will form a single timeseries.
  • AGGREGATE
    • This setting causes CDAP to return the total aggregated since the application was deployed
    • Or—in the case of dataset metrics—since a dataset was created.

Please see the CDAP Metrics documentation for more details.
